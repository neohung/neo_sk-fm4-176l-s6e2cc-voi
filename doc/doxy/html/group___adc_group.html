Detailed Description

Provided functions of ADC module:

  • Adc_Init()
  • Adc_DeInit()
  • Adc_Enable()
  • Adc_Ready()
  • Adc_EnableWaitReady()
  • Adc_Disable()
  • Adc_SwTriggerScan()
  • Adc_StopScanRepeat()
  • Adc_SwTriggerPrio()
  • Adc_ForceStop()
  • Adc_EnableIrq()
  • Adc_DisableIrq()
  • Adc_GetIrqFlag()
  • Adc_ClrIrqFlag()
  • Adc_GetStatus()
  • Adc_ReadScanFifo()
  • Adc_ClrScanFifo()
  • Adc_GetScanChannel()
  • Adc_GetScanDataValid()
  • Adc_GetScanDataCause()
  • Adc_ReadPrioFifo()
  • Adc_ClrPrioFifo()
  • Adc_GetPrioChannel()
  • Adc_GetPrioDataValid()
  • Adc_GetPrioDataCause()

Adc_Init() initializes an ADC instance according the configuration of the structure type stc_adc_config_t. Adc_DeInit() de-initializes an ADC instance.

For using the ADC instnace Adc_Enable() can be used first to enable ADC operation.

The interrupt can be configured in Adc_Init() first, after initialization, Adc_EnableIrq() is used to enable ADC interrupt selected by the structure type stc_adc_irq_sel_t. Adc_DisableIrq() is used to disable ADC interrupt also selected by the structure type stc_adc_irq_sel_t.

Adc_SwTriggerScan() and Adc_SwTriggerPrio() can be used for software trigger start for scan and priority conversion.

Adc_StopScanRepeat() stops ADC scan conversion with repeat mode.

Adc_ForceStop() stops both ADC scan and priority conversion forcely.

Adc_GetIrqFlag() can get the interrupt flag in the interrupt-polling mode and Adc_ClrIrqFlag() can clear the interrupt flag. The interrupt type can be selected by the enumeration type en_adc_irq_t.

Adc_GetStatus() return the status of ADC selected by the structure type en_adc_status_t.

Adc_ClrScanFifo() and Adc_ClrPrioFifo() clear the FIFOs.

Adc_ReadScanFifo() and Adc_ReadPrioFifo() read out the FIFOs.

Adc_GetScanChannel() and Adc_GetPrioChannel() extracts channel information from the FIFO data.

Adc_GetScanDataValid() and Adc_GetPrioDataValid() extracts data valid information from the FIFO data.

Adc_GetScanDataCause() and Adc_GetPrioDataCause() extracts trigger cause information from the FIFO data.

Note:
The callback arguments for the scan and priority result point to the according FIFO registers of the recent instance. The application must take care of emptying these FIFOs. Either the FIFO depth of the configuration can be used for counting or the function Adc_GetStatus() can be used to check if the FIFO is empty or not.
